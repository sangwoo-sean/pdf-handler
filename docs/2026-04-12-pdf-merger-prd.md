# PDF Merger — PRD (Product Requirements Document)

> MVP 데스크톱 애플리케이션: 여러 PDF 파일을 원하는 순서로 병합하여 하나의 PDF로 추출

## 1. 개요

### 1.1 문제 정의

여러 PDF 파일을 하나로 합쳐야 하는 상황이 빈번하지만, 기존 도구들은 온라인 업로드(보안 우려), 유료 구독, 또는 복잡한 UI를 요구한다.

### 1.2 솔루션

로컬에서 동작하는 가볍고 심플한 데스크톱 앱. 파일을 추가하고, 순서를 정하고, 병합 버튼 하나로 완료.

### 1.3 대상 사용자

PDF를 자주 다루는 일반 사용자 (문서 작업자, 학생, 직장인 등)

### 2.1 버전 선정 원칙

- **안정성 우선** — 최신 버전보다 생태계 호환성이 검증된 버전 선택
- **캐럿(^) 버저닝** — 마이너/패치 업데이트만 허용, 브레이킹 체인지 방지
- **네이티브 의존성 최소화** — pdf-lib은 순수 JS라 크로스 플랫폼 빌드 문제 없음

## 3. 기능 요구사항

### 3.1 파일 추가

- "파일 추가" 버튼 클릭 시 OS 네이티브 파일 선택 다이얼로그 열림
- `.pdf` 확장자만 선택 가능하도록 필터 적용
- 복수 파일 동시 선택 가능
- 추가된 파일은 목록 맨 아래에 순서대로 추가

### 3.2 파일 목록

- 각 항목에 표시: 순번, 파일명, 페이지 수
- 파일이 없을 때 빈 상태 안내 메시지 표시
- 목록 하단에 전체 페이지 수 합계 표시

### 3.3 순서 변경

- 각 항목에 ▲(위로) / ▼(아래로) 버튼
- 첫 번째 항목: ▲ 비활성화
- 마지막 항목: ▼ 비활성화

### 3.4 파일 삭제

- 각 항목에 ✕ 버튼으로 목록에서 제거
- 별도 확인 다이얼로그 없이 즉시 제거 (원본 파일에 영향 없음)

### 3.5 PDF 병합

- "PDF 병합하기" 버튼 클릭 시 OS 저장 다이얼로그 열림
- 기본 파일명: `merged.pdf`
- 목록 순서대로 모든 페이지를 하나의 PDF로 합침
- 병합 중 로딩 인디케이터 표시
- 완료 시 성공 알림
- 에러 발생 시 에러 메시지 표시

### 3.6 유효성 검사

- 파일이 2개 미만일 때 병합 버튼 비활성화
- 손상된 PDF 파일 추가 시 에러 메시지 표시 후 해당 파일 추가 거부

## 4. UI 설계

단일 화면 (Single View). 별도의 페이지 라우팅 없음.

```
┌──────────────────────────────────────┐
│  PDF Merger                     [─□✕]│
├──────────────────────────────────────┤
│                                      │
│  [+ 파일 추가]                        │
```

### 4.2 디자인 원칙

- 화이트 배경, 미니멀 스타일
- 시스템 폰트 사용 (OS 네이티브 느낌)
- 버튼은 명확한 라벨, 충분한 클릭 영역
- 색상: 주요 액션 버튼만 파란색 계열, 나머지 회색/흰색

### 4.3 윈도우

- 기본 크기: 480 x 600px
- 최소 크기: 400 x 500px
- 리사이즈 가능

## 5. 아키텍처

### 5.1 프로세스 구조

```
┌─ Main Process (Electron) ─────────┐
│  - 파일 다이얼로그 (열기/저장)       │
│  - 파일 시스템 읽기/쓰기            │
│  - PDF 병합 처리 (pdf-lib)         │
└────────────┬──────────────────────┘
             │ IPC (invoke/handle)
┌────────────┴──────────────────────┐
│  Renderer Process (React)         │
│  - UI 렌더링                       │
│  - 파일 목록 상태 관리              │
│  - 사용자 인터랙션 처리             │
└───────────────────────────────────┘
```

### 5.2 IPC 채널

| 채널 | 방향 | 설명 |
|------|------|------|
| `dialog:open-files` | Renderer → Main | 파일 선택 다이얼로그 열기 |
| `pdf:get-page-count` | Renderer → Main | PDF 파일의 페이지 수 조회 |
| `pdf:merge` | Renderer → Main | PDF 병합 실행 + 저장 다이얼로그 |

## 6. 데이터 모델

```typescript
interface PdfFile {
  id: string          // 고유 식별자 (crypto.randomUUID)
  name: string        // 파일명
  path: string        // 절대 경로
  pageCount: number   // 페이지 수
}

// 렌더러 상태
interface AppState {
  files: PdfFile[]    // 순서가 보장된 배열
  isMerging: boolean  // 병합 진행 중 여부
}
```

## 7. MVP 범위 외 (향후 고려)

다음 항목들은 MVP에서 **제외**한다:

- 개별 페이지 단위 선택/재배열
- PDF 페이지 미리보기/썸네일
- 드래그 앤 드롭 (파일 추가 및 순서 변경)
- 다국어 지원 (i18n)
- 자동 업데이트 (auto-updater)
- PDF 암호 해제
- PDF 압축/최적화
- 다크 모드

## 8. 성공 기준

- macOS, Windows에서 정상 빌드 및 실행
- 2개 이상의 PDF를 원하는 순서로 병합하여 1개의 PDF로 저장
- 병합된 PDF의 모든 페이지가 원본과 동일하게 보존
- 100MB 이하 PDF 파일 처리 가능
